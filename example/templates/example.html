navigation

navigation contains render_navbar, navtagitem, horizon_nav, vertical_nav, render_footer.

  1. render_navbar

    Usage:

    {% load navigation %}

    {% render_navbar settings.NAVBAR %} 

    其中settings.NAVBAR是定义在settings中的变量，也可以是其他地方定义的，只要页面能取到。
		格式为：[{"name": u"首页", "url": "/"},...]

  2. navtagitem

    {% load navigation %}

    {% navtagitem "手动添加水平导航" "/" %} 

    第一个参数是名称，第二个参数是url

  3. horizon_nav.

    {% load breadcrumb %}

    {% add_crumb "test" "/" %}

    {% render_breadcrumbs %} 

    test是显示的名称，"/"是url

  4. vertical_nav

    {% load navigation %}

    {% vertical_nav "" settings.HORIZION_SECTION %} 

    第一个参数是选中的名称，第二个参数是所有垂直导航项
		格式为：[{"name": u"首页", "url": "/"},...]

  5. render_footer

    {% load navigation %}

    {% render_footer %} 

    需要在settings中配置FOOTER格式为：[[{"name": u"问题反馈", },
						   {"name": u"常见问题解答", "url": "/"},...
				                 ]]
				                 	
    字典中的第一个值为列名

breadcrumb

breadcrumb contains breadcrumb, add_crumb.

  1. breadcrumb

    {% load breadcrumb %}

    {% breadcrumb settings.HORIZION_SECTION request %} 

    第一个参数的格式为：[{"name": u"首页", "url": "/"},...]

  2. add_crumb

    {% load breadcrumb %}

    {% add_crumb "test" "/" %}

    {% render_breadcrumbs %} 

    test是显示的名称，"/"是url

pagination

  1. pagination

    {% load pagination %}

    {% pagination pages "/" request %} 

    第一个参数是page对象，第二个参数是分页url前缀

bootstrap

  1. bootstrap

    bootstrap 将form表单转化为bootstrap样式，不支持继承ModelForm生成的form.

    {% load bootstrap %}

    {{ form|bootstrap }}

form

form 重写form表单，在页面上展现bootstrap样式.增加了MarkdownField，支持markdown输入框

  1. form

    from html5tags.bootstrap import form

    class TestForm(form.Form): bootstrap_textarea = floppy.MarkdownField()

    其他field使用方法和django的form相同

    页面上渲染可以使用{{ form.as_bootstrap}}

可以自定义form的field显示

{% load render_form_field %}

{% for field in form %}

{% render_field field "div" "form-group" "label" "" %}
